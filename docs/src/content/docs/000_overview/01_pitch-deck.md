## ビジョン（Why now）

- “調整の往復”が価値を生まない時間。初回回答率の最大化でリードタイムを短縮する。
- Doodleが確立した\*\*三択グリッド（◯/△/×）\*\*は意思決定を高速化。semdleはこれを標準UIとする。
- 匿名×CAPTCHAで「参加は最短ワンクリック」、主催は Google/Magic Link で安全に昇格できる時代背景（SaaS/ID連携の成熟）。
- 締切＋自動リマインダで“決めどき”を作る。通知依存を最小化し、心理的負担を下げる。
- モバイル優先のWeb体験（DoodleもWeb中核）。アプリ依存ではなくレスポンシブで完結。

## エレベーターピッチ（Who / What / Benefit）

- Who: 社内外の参加者を巻き込む日程調整が多いビジネスパーソン。
- What: semdleはリンクを開いて◯/△/×をタップするだけ。主催は Google / Magic Link で作成・編集、参加者リストの非表示や回答上限などのコントロールも可能。
- Benefit: 初回回答率↑／決定までの時間↓／誤解の源（TZ差や不透明な優先度）↓。

## 対象ユーザー（ペルソナ：1名）

- 名前: 中村 直樹（35）／営業。社外ゲストが多く、ITリテラシーは中程度。
- 目標: 1往復で枠を確定。相手の負担を増やさず、進行をコントロール。
- 制約: ゲストにアカウント作成やアプリ導入を求めたくない。
- 成功の定義: 半日以内に過半の回答が集まり、最適枠が自明になっている。
- 隠れた不安: 参加者一覧の公開での心理的障壁（匿名・非表示で緩和）。 ￼

## 顧客のJobs To Be Done と主要ペイン（トップ3）

- JTBD: 複数候補から最小の摩擦で合意を取りたい。
- ペイン1: ログイン要求や長いフォームで離脱。
- ペイン2: 比較困難 "◯/△/×" が混在しても“どれがベターか”が直感的でない。
- ペイン3: 公開範囲の不安—参加者名の開示や過度な情報共有が嫌。→非表示トグルで緩和。

## 成功指標（KPI）と目標値（主要指標＋補助指標）

### 主要KPI

- 初回回答率（リンク開封→1回答）: ≥70%（2週間β）
- 決定TAT（作成→最適枠確定）: ≤24hで80%達成（締切＋自動リマインド前提）
- モバイル完了率: モバイルでの全体回答の≥60%

### 補助指標

- 中断率（滞在<10秒・無回答）: <15%
- 不一致率（TZ誤解による修正）: <1%（自動TZ）
- NPS（内向け）: ≥+30

## ソリューション概要（コアユーザーフロー）

1. 作成: 主催はGoogle/Magic Linkでイベント作成、候補を入力。Yes/If-need-be/Noで投票前提の枠設計。
2. 共有: URLを送付。ゲストはページ表示と同時に匿名サインイン＋CAPTCHAで即投票可。
3. 回答: グリッドUIで◯/△/×をタップ。投票上限や参加者非表示は主催者がON/OFF可能。
4. 合意: 締切＋自動リマインドで未回答を掘り起こし、最終枠を確定。
5. 同期: カレンダー連携で確定枠を各自のカレンダーへ配信。TZは自動調整。

## MVPスコープ（Must / Should / Could、Mustは最大3つ）

### Must

- 三択グリッド（Yes/If-need-be/No）と一括集計（候補別の◯/△/×カウント）
- 匿名即投票＋CAPTCHA＋TZ自動（表示はローカル）
- 参加者非表示（トグル）と投票上限（荒らし防止・心理安全）

### Should

- 候補スコアリング（Yes=2/Maybe=1）
- 締切の設定と自動リマインド（簡易）
- カレンダー招待（Google Calendar の最小接続）

### Could

- Privateイベント（招待コード）
- 締切到来後の自動確定提案
- 候補数の上限ガイド（6個以内推奨のUXヒント）

## リスク＆検証計画（上位3つのリスク＋最初の3つの実験）

### リスク

- 乱投票（匿名の悪用）→ 上限制限・CAPTCHA・締切で牽制。
- TZ誤解での不一致→ 自動TZと明示表示を徹底。
- 設定の過多で初回離脱→ 候補6件以内ルール・設定は後置化。

### 実験（KPI/期間）

- 初回回答率A/B（2週間）: 匿名対ログイン必須。目標≥70%。
- 締切＋リマインド実装効果（10日）: 決定TATが≤24hに収束するか。 ￼
- TZ誤解率（2週間）: 説明文の有無で不一致率\*\*<1%\*\*を維持。 ￼

## Go-to-Market（初期チャネル＋最初のローンチ施策）

- 初期チャネル: 社内Slack
- 施策
  - 「1往復で会議が決まる」GIFデモ（モバイル操作）
  - 回答催促の自動文面テンプレ（“明日12:00で締切”）
  - 成功事例共有（半日で決まった案件）
- 競合意識: When2Meetの色分けUIは学習コストが低いが、semdleは三択グリッドで決定スピードを重視。
